Лёгкие:

Пользователь пишет «Напомни через 20 минут выключить чайник» или «Напомни завтра в 9:00». Бот парсит время (простым паттерн-матчингом), сохраняет задачу в БД и отправляет пуш/сообщение по таймеру.

Бот задает пользователю 5-7 вопросов с кнопками (или сбором свободного текста), формирует из ответов структурированный JSON и отправляет его в CRM (например, amoCRM или Bitrix24) или сохраняет в таблицу.

Классическая база знаний. Пользователь пишет вопрос, бот ищет совпадения по ключевым словам (или использует простую косинусную близость без LLM) и выдает готовый, заранее заготовленный ответ с картинкой или ссылкой.

Пользователь задает вопрос или просит выбрать вариант (например, «Куда пойти в обед?»). Бот использует встроенный генератор случайных чисел, чтобы выдать один из заранее загруженных в массив вариантов.

Пользователь отправляет текст, а бот применяет к нему жесткие правила: переводит в верхний регистр, убирает все гласные, переводит текст в азбуку Морзе или считает количество символов/слов.



Средние:

Пользователь описывает проблему своими словами. LLM анализирует текст, определяет категорию (биллинг, техподдержка, жалоба), оценивает токсичность/эмоциональность и, в зависимости от результата, маршрутизирует диалог нужному оператору или генерирует черновик ответа.

LLM выступает в роли строгого, но вежливого репетитора. Пользователь пишет на изучаемом языке, а бот не только поддерживает диалог, но и в скрытом режиме (или в конце сообщения) указывает на грамматические ошибки и предлагает правильные варианты.

Пользователь задает параметры: «Напиши 3 идеи для поста в Telegram про кибербезопасность, тон — ироничный, целевая аудитория — IT-специалисты». LLM генерирует контент. Можно добавить кнопку «Сделать более формальным», которая меняет системный промпт на лету.

Пользователь пишет: «Ищу ноутбук для монтажа видео до 100к». LLM извлекает сущности (бюджет=100000, задача=монтаж), формирует структурированный JSON и передает его в простую функцию-поиск по каталогу товаров, после чего красиво оформляет выдачу.

Пользователь скидывает «простыню» текста (статью, транскрибацию часового созвона). Бот сжимает её до 5 буллитов, выделяет ключевые решения и задачи (Action Items) с указанием ответственных.



Сложные:

Сотрудник спрашивает: «Сколько у меня дней отпуска и могу ли я взять неделю с понедельника?».
Шаг 1 (RAG): Бот ищет в регламентах компании правила предоставления отпусков.
Шаг 2 (БД): Делает запрос в HR-систему (например, 1С:ЗУП) через API, чтобы узнать точный баланс дней сотрудника.
Шаг 3 (LLM + Логика): Сопоставляет правила и баланс, отвечает пользователю.
Шаг 4 (Action): Если сотрудник согласен, бот сам создает заявку на отпуск в корпоративном портале.

Система состоит из трех скрытых агентов.
Агент 1 (L1): Общается с клиентом, использует RAG по базе знаний для решения типовых проблем. Если не может — эскалирует.
Агент 2 (L2): Получает контекст, делает запрос в API мониторинга (проверяет статус серверов/лицевых счетов), анализирует логи.
Агент 3 (L3): Если проблема уникальна, формирует подробный баг-репорт и создает тикет в Jira, а клиенту сообщает сроки.

Клиент банка пишет: «Почему у меня списали деньги в этом месяце больше, чем в прошлом, и на чем я могу сэкономить?».
Бот через LLM генерирует сложный SQL-запрос к транзакциям клиента.
Выполняет запрос к БД, получает сырые данные.
Второй LLM анализирует цифры, сравнивает с категориями трат и генерирует персональный финансовый совет.

Бот ведет нового сотрудника по программе адаптации на протяжении месяца.
Использует RAG для ответов на вопросы «А как оформить пропуск?».
Хранит в БД прогресс сотрудника (какие курсы прошел, какие чек-листы закрыл).
На основе прогресса LLM генерирует персональные ежедневные задания.
В конце недели проводит интерактивное тестирование, где LLM выступает в роли экзаменатора, оценивая ответы и адаптируя сложность следующих вопросов.

Менеджер пишет боту: «Нужно сделать КП для компании Ромашка, они хотят купить 50 серверов, у них скидка 10% и отсрочка платежа».
Бот через API находит карточку клиента в CRM и проверяет его кредитный лимит.
Через RAG ищет актуальные прайсы и условия поставки серверов.
LLM рассчитывает финальную стоимость с учетом всех условий.
Генерирует красивый PDF-документ (через интеграцию с шаблонизатором) и отправляет его на почту клиенту.
